Rust's share of the kernel

The whole mainline tree is about 30.1M lines of code across 44 languages — overwhelmingly C and C headers. Rust is still a sliver, but a fast-growing one: it went from 0% in 6.0 (no .rs at all) to 0.019% when it first landed in 6.1, to 0.31% in 7.1.

0.31% of the kernel's 30.1M SLOC is Rust (93,245 of 30,101,072 lines). Of that, ~47.0k is vendored crates; first-party kernel Rust is 46.3k (0.154%).

6.10.14 54 files · 5,642 SLOC · 421.2 KiB

Why totals fall here — a one-time cleanup, not a retreat from Rust. 6.10 removed the in-tree rust/alloc fork: the kernel's private copy of Rust's standard alloc library (Vec, Box, slice, raw_vec, …) — ~4,360 SLOC across 14 files, counted here under Vendored crates — and replaced it with thin allocator abstractions in rust/kernel/alloc/ (box_ext, vec_ext). First-party kernel Rust kept growing straight through the drop: the Kernel crate went 3,812 → 4,057 → 4,845 SLOC across 6.9 → 6.10 → 6.11. Only the vendored standard-library copy went away, so the total fell even as the kernel's own Rust expanded.

Total Rust leaps from 36,225 SLOC (6.18) to 89,336 — but almost all of that is vendored crates: third-party libraries from the Rust ecosystem (crates.io) copied verbatim into rust/, because the kernel build is self-contained and can't fetch dependencies over the network. In 6.19 the standard procedural-macro toolchain was vendored in one go — syn (41,716 SLOC, a full Rust-source parser), proc-macro2 (3,782) and quote (1,460): 46,958 SLOC across 75 files — so the kernel's own macros (rust/macros/) can lean on them instead of hand-rolled parsing. This is external tooling, not kernel logic: set it aside and first-party kernel Rust grew steadily, 36,225 → 42,378 SLOC (6.18 → 6.19). The hero chart breaks it out as its own "Vendored crates" band for exactly this reason.

Methodology

How the numbers are produced — fully reproducible from the scripts in the repository.

For each kernel series ≥ 6.0 the latest patch release is resolved from https://cdn.kernel.org/pub/linux/kernel, the source tarball downloaded and only *.rs files extracted. File sizes come from the extracted sources; line counts (code / comment / blank) from CLOC tool v1.98. Generated bindings that are produced only at build time are not shipped in the tarball and therefore not counted. The share-of-kernel figures come from a second cloc pass over the entire extracted tree (all languages), so the denominator is the whole kernel's SLOC measured the same way as the Rust numerator. Data regenerated with scripts/build.sh; machine-readable results live in data/kernels.json.
